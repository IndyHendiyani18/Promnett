Pengertian GIT

GIT adalah sebuah version control system yang telah digunakan oleh para developer untuk dapat mengembangkan software secara bersamaan yang digunakan oleh pengembang dan programmer sebagai sistem pengendalian untuk mengelola proyek pengembangan perangkat lunak. Tujuan utama penggunaan GIT adalah untuk mengatur berbagai versi dari kode sumber program dengan menentukan baris dan kode yang akan dimasukkan atau diubah. GIT termasuk dalam kategori DVCS (Distributed Version Control System), yang berarti kode tersebut tidak hanya memiliki satu lokasi penyimpanan untuk riwayat lengkap perangkat lunak. Setiap salinan dari kode yang digunakan oleh pengembang juga berfungsi sebagai repository yang dapat menyimpan riwayat lengkap dari semua perubahan yang terjadi.

Perbedaan GIT dan GitHub

Antara GIT dan GitHub, keduanya merupakan dua platform yang didirikan oleh perusahaan dengan tujuan yang serupa, tetapi mereka memiliki perbedaan dalam fitur yang mereka tawarkan. Kedua platform ini sangat berguna bagi para programmer dalam kerja tim untuk mengelola kode skrip. Dengan menggunakan sistem pengendalian versi, semua pekerjaan dapat dengan mudah dipantau dan dievaluasi.

Perintah Dasar GIT

  • GIT init: untuk membuat sebuah repository yang berada pada file lokal dimana nantinya file tersebut bernama .git.
  • GIT status: untuk mengetahui status dari sebuah repository lokal.
  • GIT add: untuk menambahkan file baru yang ada pada repository yang telah dipilih.
  • GIT commit: untuk menyimpan perubahan yang telah dilakukan.
  • GIT push: untuk mengirimkan perubahan file setelah di commit ke remote repository.
  • GIT branch: dapat melihat seluruh branch yang ada dalam repository.
  • GIT checkout: dapat menukar branch yang aktif dengan branch yang telah dipilih.
  • GIT merge: untuk menggabungkan branch yang aktif dan yang telah dipilih.
  • GIT clone: dapat membuat salinan repository lokal.
